Local Storage Vs Session Storage

In this exercise, we will build a click counter first using local storage, then a second counter using session storage to compare.

Local Storage

Each time that the button is clicked, the click counter will increase.

If you refresh the page and click, you'll see that your clicks carried over.

Close this window and run the application again, you'll see that your clicks still did not reset.

your clicks are being stored locally, so no matter what you do your clicks will only go up from here.

Session Storage

Similarly to local storage, each time that the button is clicked, the click counter will increase.

If you refresh the page and click, you'll see that your clicks also carried over.

However, if you close the window and rerun the application, you'll see that your clicks started over.

Session storage will persist as long as the session is still taking place. When the window is closed, the session ends.

Session storage is very useful when storing user information and decisions, since once the user signs out the session ends and all sensitive info disappears with it.
